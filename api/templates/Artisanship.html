Fullstack Developer for Streamline

Invoice Class

  • Intermediary object between inputs (CSV, TSV, Excel, GUI Form, and PDF) and XML output
  • The object class was easily converted into JSON, HTML, XML, etc
  • Allowed for flexibility, asserted necessary conditions, and made it easy to convert to XML from many different input types
  • Invoice ID creator within Invoice Class


PDF to Invoice Functionality

  • Planned the usage of AWS Textract in my week 8 blog
  • Used AWS Textract for the first time: Very accurate OCR techniques
  • Immediate appreciation due to the simplicity of usage
    • Textract's 'analyze_expense' function returned a easy to traverse dictionary
    • Fields were automatically interpreted and passed in a key-value pair

Rendering Functionality

  • Flask! Very easy to pass in HTML and CSS to render a document beautifully
  • Reinforced the benefit of creating the Invoice Class
    • Extremely easy to add an additional toHTML() function within the Tag and Attribute parent classes
    • Formed the basis of the render functionality


Frontend Authentication

  • Planned the usage of OAuth 2.0 and Google APIs in my week 8 blog
  • OAuth 2.0 and Google APIs allow easy sign on with Google, also allowing them to stay logged in
    • No passwords / usernames required
    • Reduces risk of security issues, and the user knows explicitly what the application accesses
  • The user's data (Profile Photo and ID) are stored within an AWS S3 bucket

Frontend Access to Cloud Storage

  • Flask! Allowed very easy checking to retreive user's ID if they are authenticated
  • OAuth 2.0 Authentication led to easily implementable Cloud Storage, due to Google IDs
    • No need to store / validate IDs
    • Comfortably integrates with AWS S3 storage
  • Pre-exisitng setup of AWS S3 bucket (Done by team member) allowed easy storage, collection and deletion of data
